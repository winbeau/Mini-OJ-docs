# 09 · 第12章 多线程判题队列(里程碑 M6a)

> 目标:提交进队列、后台工作线程并发判题;用线程做 **TLE 超时**;界面用 `SwingWorker` 不卡。

## 知识点(大纲对照)

| 要点 | 处理 |
|---|---|
| `Thread`/`Runnable`、启动/`run` | **动手** |
| 线程同步 `synchronized` | **动手**(队列) |
| 协调同步 `wait`/`notify` | **动手**(空队列等待) |
| 线程联合 `join`(含带超时) | **动手**(TLE) |
| 守护线程 `setDaemon` | **动手**(工作线程) |
| GUI 线程 `SwingWorker`/EDT | **动手**(不卡界面) |
| 计时器线程 `Timer`、优先级 | **了解即可**,join-timeout 已够做 TLE |

## 1. 线程安全队列 `oj/judge/queue/JudgeQueue.java`(synchronized + wait/notify)

```java
package oj.judge.queue;
import oj.core.Submission;
import java.util.*;
public class JudgeQueue {
    private final Queue<Submission> q = new LinkedList<>();   // LinkedList 不在大纲,作队列用;懂原理即可

    public synchronized void put(Submission s) {   // synchronized:同一时刻只一个线程改队列
        q.add(s);
        notifyAll();                                // 唤醒在 take() 里等待的工作线程
    }
    public synchronized Submission take() throws InterruptedException {
        while (q.isEmpty()) wait();                 // 队列空 → 释放锁并等待
        return q.poll();
    }
}
```
> 现实里直接用 `java.util.concurrent.BlockingQueue` 更省事,但本章目的是**亲手体会 synchronized/wait/notify**,所以手写。

## 2. 带超时的判题(join 超时 + 守护线程做 TLE)

把"跑一次判题"放进子线程,主线程 `join(timeLimit)` 等;超时就判 TLE:
```java
package oj.judge;
import oj.core.*;
public class TimedJudge {
    public static JudgeResult judgeWithLimit(Judge judge, Problem p, Solution s) {
        final JudgeResult[] box = new JudgeResult[1];
        Thread t = new Thread(() -> box[0] = judge.judge(p, s));
        t.setDaemon(true);                          // 守护线程:主程序退出它不拦着
        t.start();
        try {
            t.join(p.getTimeLimitMs());             // 最多等 timeLimit 毫秒
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
        if (t.isAlive()) {                          // 还没判完 → 超时
            t.interrupt();
            return new JudgeResult(Status.TLE, 0, p.getCases().length, "超时", p.getTimeLimitMs());
        }
        return box[0];
    }
}
```
> 说明:`interrupt()` 只能中断会响应中断的代码;对"死循环不检查中断"的解法无法真正掐断(教学版接受这个局限,标注即可)。这里也正好用上了 M3 定义的 `TimeLimitException` 概念。

## 3. 工作线程 `oj/judge/queue/JudgeWorker.java`(Runnable + 守护线程)

```java
package oj.judge.queue;
import oj.core.*;
import oj.judge.*;
import oj.db.SubmissionDao;
import java.util.Map;
public class JudgeWorker implements Runnable {
    private final JudgeQueue queue;
    private final Map<Integer, Problem> problems;       // 题号→题目
    private final Map<Integer, Solution> solutions;     // 提交id→解法(教学版内存暂存)
    private final Judge judge = new StandardJudge();
    private final SubmissionDao dao = new SubmissionDao();

    public JudgeWorker(JudgeQueue q, Map<Integer,Problem> ps, Map<Integer,Solution> sols) {
        this.queue = q; this.problems = ps; this.solutions = sols;
    }
    @Override public void run() {
        while (true) {
            try {
                Submission sub = queue.take();                       // 阻塞取
                Problem p = problems.get(sub.getProblemId());
                Solution s = solutions.get(sub.getId());
                JudgeResult r = TimedJudge.judgeWithLimit(judge, p, s);
                sub.setResult(r);
                dao.save(sub);                                       // 入库(复用 M5b)
                System.out.println("判完 #" + sub.getId() + " -> " + r);
            } catch (InterruptedException e) { break; }
            catch (Exception e) { System.err.println("判题出错: " + e.getMessage()); }
        }
    }
}
```
启动 N 个守护工作线程:
```java
JudgeQueue queue = new JudgeQueue();
for (int i = 0; i < 4; i++) {
    Thread t = new Thread(new JudgeWorker(queue, problems, solutions));
    t.setDaemon(true);
    t.start();
}
// 提交:queue.put(submission);  多个提交会被 4 个线程并发判
```

## 4. GUI 不卡(SwingWorker)

把 M5c 的 `onSubmit` 改成后台判、回 EDT 更新:
```java
new javax.swing.SwingWorker<JudgeResult, Void>() {
    protected JudgeResult doInBackground() {                 // 后台线程判题
        return TimedJudge.judgeWithLimit(new StandardJudge(), p, s);
    }
    protected void done() throws Exception {                 // 回到 EDT 更新界面
        JudgeResult r = get();
        fillResult(p, r);
        JOptionPane.showMessageDialog(OjFrame.this, r.toString());
    }
}.execute();
```

## 验收标准

- [ ] 一次性 `put` 多个提交,4 个工作线程能**并发**判(打印交错)。
- [ ] 死循环/超慢解法被判 **TLE**(`join` 超时生效),程序不卡死。
- [ ] 队列用 `synchronized`+`wait/notify` 实现,空队列时工作线程阻塞而非空转。
- [ ] GUI 判题期间界面不冻结(`SwingWorker`)。
- [ ] 工作线程是守护线程,主程序结束能正常退出。
