# Mini-OJ 总设计(全局契约)

> 这份是**地基**:定下全项目的包结构、类/接口清单、核心数据模型签名。各章文档都引用本文,**不得擅自改名或改签名**。
> 原则:**简单、教学优先**。能少一个类就少一个,能不实现的非重点就不实现。

## 1. 简化原则(我们故意不做什么)

- **不真编译运行用户源码**:用 `Solution` 接口模拟"用户提交的程序"。学生写一个实现类就是一次提交。
  - 副作用:`CE`(编译错误)在核心里不会自然出现 → 只在**可选拓展**(`ProcessBuilder` 跑外部程序)里提一句,核心状态以 `AC/WA/RE/TLE/PE` 为主。
- **不引入框架**:全程 `javac`/`java` 手撸;M5b 接 MySQL 驱动时才用 `java -cp` 挂 jar(不强制上 Maven)。
- **非重点只了解不实现**:位运算、`instanceof` 细节、Derby/Access/SQLServer、读写图像、文件锁、颜色对话框等——文档里标注"了解即可",不写进项目。

## 2. 关键设计决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 用户解法表示 | `Solution` 接口:`String solve(String input)` | 纯 Java,免去外部进程;天然教接口+多态 |
| 判题结果状态 | `enum Status` | 最干净(enum 不在大纲但极简单;也可用 `final String` 常量替代,呼应 Ch5 常量) |
| 判题器演进 | M2 具体 `SimpleJudge` 类 → M3 重构成 `Judge` 接口 + `AbstractJudge` + 子类 | "学到抽象再抽象",重构即教学 |
| 题库来源 | M2 硬编码 → M5a 改读 `problems/` 文件 → M5b 元数据进 MySQL | 一步步替换,旧代码尽量复用 |
| TLE | M3 先定义异常类(学语法)→ M6a 用线程真正触发 | 概念与实现分两阶段 |

## 5. 核心数据模型契约(签名锁定,后续章节照此实现)

> 这些是稳定接口,M2 定下后基本不变。字段尽量 `final` + 私有 + getter。

```java
// oj.core.Status  —— 全项目唯一权威定义
enum Status { AC, WA, TLE, MLE, RE, CE, PE }
// 谁产生谁:Java 内判题(M3/M4)只产 AC/WA/RE(PE 由可选 FormatChecker 产);
//          判题机(第11章)产 AC/WA/TLE/MLE/RE/CE,其内部错 ERR 由 MachineJudge 映射成 RE。

// oj.core.TestCase
class TestCase {
    private final String input;
    private final String expected;
    TestCase(String input, String expected)
    String getInput();  String getExpected();
}

// oj.core.Problem
class Problem {
    private final int id;
    private final String title;
    private final TestCase[] cases;
    private final long timeLimitMs;                 // M6a 才真正用到
    Problem(int id, String title, TestCase[] cases)                 // 默认时限 1000ms
    Problem(int id, String title, TestCase[] cases, long timeLimitMs)   // 重载(Ch4)
    int getId(); String getTitle(); TestCase[] getCases(); long getTimeLimitMs();
}

// oj.core.JudgeResult
class JudgeResult {
    private final Status status;
    private final int passed, total;
    private final String detail;     // 失败说明,如 "case#2 期望 5 实际 4"
    private final long elapsedMs;
    JudgeResult(Status status, int passed, int total, String detail, long elapsedMs)
    Status getStatus(); int getPassed(); int getTotal(); String getDetail(); long getElapsedMs();
    boolean isAccepted();            // status == AC
    @Override String toString();     // 如 "AC 3/3 (12ms)"
}

// oj.core.Submission
class Submission {
    private static int counter = 0;          // 类变量:累计提交数(Ch4 static)
    private final int id;                     // 由 counter 自增得到
    private final int problemId;
    private final String userName;
    private JudgeResult result;               // 判完回填
    private final LocalDateTime time;         // M4 引入
    Submission(int problemId, String userName)
    int getId(); int getProblemId(); String getUserName(); LocalDateTime getTime();
    JudgeResult getResult(); void setResult(JudgeResult r);
    static int count();                       // 返回 counter
}

// oj.judge.Solution  (M2 先做成普通类,M3 改成接口)
interface Solution {
    String solve(String input);              // 输入一组数据 → 产生输出
}

// oj.judge.Judge  (M3)
interface Judge {
    JudgeResult judge(Problem problem, Solution solution);
}
```

## 6. 命名 / 风格约定

- 包名小写 `oj.xxx`;类名大驼峰;方法/变量小驼峰;常量全大写。
- 缩进 4 空格;采用 K&R(左括号不换行)。每个 public 类/方法写一句 Javadoc。
- 一个文件一个 public 类;`main` 只放在 `Main`/各 `*Server`/`*Client`。
