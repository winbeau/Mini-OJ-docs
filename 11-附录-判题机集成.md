# 11 · 附录 判题机集成(Java 调外部判题机)

> 前面 M3/M4 的 Java 判题(`Judge` 接口 / `AbstractJudge` / `Solution`)是**讲 OOP 的教学骨架,保留不动**。
> 本章新增一条**真实判题通路**:Java 通过 `ProcessBuilder` 调用独立的「判题机」(`judge/`),去编译运行真实的 **C++/Python 提交源码**。
> 一句话:**OOP 怎么抽象——看 M3/M4;代码怎么真跑起来被判——看本章。**

## 判题机是什么(详见 `judge/README.md`)

- 位置:本仓 `judge/`,单文件 `judge.cpp`,**编译型**:`cd judge && make`。
- 调用:
  ```bash
  ./judge --problem <题目目录> --src <提交源码> --lang <cpp|python> [--time-ms 1000] [--mem-mb 256]
  ```
- 输出(stdout 一行 JSON):
  ```json
  {"status":"AC","passed":3,"total":3,"time_ms":3,"mem_kb":1840,"detail":""}
  ```
- 它负责脏活:`fork`+`exec` 跑提交、`setrlimit` 限资源、墙钟看门狗抓超时、按行比对输出。
  Java 端**只管调用 + 解析结果**,简单。

## 第一步:`Status` 已含 MLE(见 00 §5)

判题机会返回 `MLE`(超内存)和 `ERR`(内部错)。`Status` 在 `00 §5` 已统一为权威定义:
```java
// oj.core.Status
public enum Status { AC, WA, TLE, MLE, RE, CE, PE }
```
判题机的 `ERR` 无对应枚举值,由下面 `MachineJudge.parse` 兜底映射成 `RE`;`PE` 判题机不产生。

## 第二步:`MachineJudge`——调用外部判题机(本章核心)

`oj/judge/MachineJudge.java`
```java
package oj.judge;
import oj.core.*;
import java.io.IOException;
import java.nio.charset.StandardCharsets;
import java.util.regex.*;

public class MachineJudge {
    private final String judgeBin;     // 判题机可执行文件, 如 "judge/judge"
    public MachineJudge(String judgeBin) { this.judgeBin = judgeBin; }

    /** 判一份真实提交源码。problemDir 形如 "problems/1";lang = "cpp" | "python" */
    public JudgeResult judge(String problemDir, String srcFile, String lang,
                             long timeMs, int memMb)
            throws IOException, InterruptedException {
        Process p = new ProcessBuilder(                      // 运行外部可执行文件(Ch10)
                judgeBin, "--problem", problemDir, "--src", srcFile, "--lang", lang,
                "--time-ms", String.valueOf(timeMs),         // 把题目时限/内存真正传给判题机
                "--mem-mb",  String.valueOf(memMb)
        ).start();
        String json = new String(p.getInputStream().readAllBytes(), StandardCharsets.UTF_8);
        p.waitFor();
        return parse(json);
    }

    /** 便捷重载:题目目录按 id 推导,时限取 Problem 自带值,内存默认 256MB。 */
    public JudgeResult judge(Problem problem, String srcFile, String lang)
            throws IOException, InterruptedException {
        return judge("problems/" + problem.getId(), srcFile, lang,
                     problem.getTimeLimitMs(), 256);
    }

    // 极简 JSON 解析(教学够用): 正则抽字段(Ch8)
    private JudgeResult parse(String json) {
        Status status;
        try { status = Status.valueOf(str(json, "status")); }
        catch (IllegalArgumentException e) { status = Status.RE; }   // ERR 等未知状态兜底
        int  passed = (int) num(json, "passed");
        int  total  = (int) num(json, "total");
        long timeMs = num(json, "time_ms");
        return new JudgeResult(status, passed, total, str(json, "detail"), timeMs);
    }
    private static String str(String json, String key) {
        Matcher m = Pattern.compile("\"" + key + "\"\\s*:\\s*\"(.*?)\"").matcher(json);
        return m.find() ? m.group(1) : "";
    }
    private static long num(String json, String key) {
        Matcher m = Pattern.compile("\"" + key + "\"\\s*:\\s*(\\d+)").matcher(json);
        return m.find() ? Long.parseLong(m.group(1)) : 0;
    }
}
```
> 注:`detail` 含转义(如 CE 的多行编译错误),正则简易解析只够"看个大概";要精确可换真正的 JSON 库——但教学版不必。

## 第三步:跑通

```java
// 先 cd judge && make 生成 judge 二进制;题目目录放好 1.in/1.out...
MachineJudge mj = new MachineJudge("judge/judge");
JudgeResult r = mj.judge("judge/examples/problems/aplusb",
                         "judge/examples/solutions/ac.cpp", "cpp", 1000, 256);
System.out.println(r);     // 期望: AC 3/3 (..ms)
```

## 与现有设计的关系(两条通路并存)

| 通路 | 判什么 | 价值 |
|---|---|---|
| **Java 内判题**(`Judge`/`AbstractJudge`/`Solution`,M3/M4) | Java 写的 `Solution` 对象 | 教接口/继承/多态/异常 |
| **外部判题机**(`MachineJudge` + `judge/`,本章) | 真实 C++/Python 源码 | 真编译真运行、真资源限制 |

两者都产出同一个 `oj.core.JudgeResult`,但**判定口径不同**:判题机只做精确匹配(详见下方影响清单),
浮点容差 / special judge 仅在 Java 内判题通路有。想再练一次多态,可让 `MachineJudge` 也实现一个统一接口
(如 `interface SubmissionJudge { JudgeResult judge(...); }`),与 Java 内判题在更高层对齐——可选。

## 集成判题机:对前面各章的影响清单

> 前面 `00–10` 是按「Java 内判题」写的。**改走判题机时**,按本清单调整对应章节(其余不变);
> 不改走判题机、纯学 Java 时,前面各章原样即可。两条通路的差异都收口在这里。

**判定口径(Status / 输出比对)**
- 判题机只做**逐行去尾空白的精确匹配**,无浮点容差、无 special judge、不产 `PE`。M3/M4 的 `SpecialJudge`(浮点 1e-6)、`FormatChecker`(PE)**只属 Java 内判题通路**,不随源码进判题机。
- 浮点题要交判题机:出题时把标准答案固定到指定小数位、并要求选手同格式输出(把"容差"前移到出题约定)。
- `Status`:判题机产 `AC/WA/TLE/MLE/RE/CE`,`ERR`→映射 `RE`,`PE` 不产。

**Submission / 数据库(M2 Ch4、M5b Ch11)**
- 用判题机时 `Submission` 要多带两项:`String lang`(cpp/python)+ 源码(`String sourceCode` 或 `String srcPath`)——否则 `MachineJudge` 无从取 `--src/--lang`。
- `submissions` 表相应加列:`lang VARCHAR(8)`、`source MEDIUMTEXT`(或 `src_path VARCHAR(255)`)。结果字段不变。

**GUI(M5c Ch9)**
- `08` 正文用内置 Java `Solution`,是「Java 内判题教学版」。判题机版界面要改:加**语言下拉**(`JComboBox`)、**源码编辑区/文件选择**(`JTextArea` 装源码 或 `JFileChooser`),`onSubmit` 落盘源码后调 `MachineJudge.judge(problem, srcFile, lang)`。

**多线程(M6a Ch12)**
- 判题机**自带 TLE**(`setrlimit`+墙钟,死循环能真掐断)。改走判题机后 `JudgeWorker` 直接调 `MachineJudge`,**不要再外套 `TimedJudge`/join 超时**(双重超时且 `interrupt()` 杀不掉外部子进程)。
- `09` 的 `TimedJudge`(Java 线程超时)只用于「Java 内判题」教学;判题机版里线程只管并发、不管超时。

**网络(M6b Ch13)**
- `10` 正文协议发"题号+解法名"、服务端用内置 `Solution`,是「Java 内判题教学版」。判题机版协议要能**承载源码**:先发一行头(题号+语言+用户名+源码字节数),再发源码正文;服务端读出写临时文件,调 `MachineJudge` 回写结果。**单行 `readLine()` 读法对多行源码不适用**,改按字节数读正文。

**题库目录**
- `--problem` 直接传 `problems/<id>` 即可:判题机只挑成对 `*.in/*.out`,**自动忽略 `title.txt`** 等,和 `06` 的目录约定兼容。
