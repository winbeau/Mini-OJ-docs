# 06 · 第10章 文件题库与序列化(里程碑 M5a)

> 目标:题目不再硬编码,改从 `problems/` 目录读;提交记录序列化落盘,重启能读回历史。

## 知识点(大纲对照)

| 要点 | 处理 |
|---|---|
| `File`:目录遍历、`exists`、属性 | **动手**(扫题库) |
| 字符输入流 + 缓冲流(`FileReader`/`BufferedReader`) | **动手**(读 .in/.out) |
| 对象流 + 序列化(`ObjectInputStream`/`Serializable`) | **动手**(存提交) |
| `Scanner` 解析文件 | **了解/可选** |
| 字节流、随机流、数据流、数组流、图像、进度条、文件锁 | **了解即可**,不实现 |

## 1. 让 Submission/JudgeResult 可序列化

序列化要求对象图上每个类都 `implements Serializable`:
```java
// Submission, JudgeResult, (若被引用)TestCase 都加:
public class Submission implements java.io.Serializable { ... }
public class JudgeResult implements java.io.Serializable { ... }
```
> `enum Status` 天生可序列化,无需改。`Solution`(逻辑/Lambda)**不要**序列化——`Submission` 只存结果数据,不存解法。

## 2. 题库加载器 `oj/io/ProblemLoader.java`(本章核心)

```java
package oj.io;
import oj.core.*;
import java.io.*;
import java.util.*;
public class ProblemLoader {
    private static final String ROOT = "problems";

    public static Problem load(int id) throws IOException {
        File dir = new File(ROOT, String.valueOf(id));
        if (!dir.isDirectory())                                // File.isDirectory
            throw new FileNotFoundException("题目不存在: " + id);
        String title = readAll(new File(dir, "title.txt")).trim();

        List<TestCase> list = new ArrayList<>();   // ArrayList 不在大纲,但比手动扩容数组简单,这里用它
        int n = 1;
        File in;
        while ((in = new File(dir, n + ".in")).exists()) {    // 按 1.in/2.in... 顺序遍历
            String input    = readAll(in);
            String expected = readAll(new File(dir, n + ".out"));
            list.add(new TestCase(input, expected));
            n++;
        }
        return new Problem(id, title, list.toArray(new TestCase[0]));
    }

    // 字符流 + 缓冲流读整个文件
    private static String readAll(File f) throws IOException {
        StringBuilder sb = new StringBuilder();
        try (BufferedReader br = new BufferedReader(new FileReader(f))) {  // try-with-resources 自动关流
            String line;
            while ((line = br.readLine()) != null) sb.append(line).append('\n');
        }
        return sb.toString();
    }
}
```

## 3. 提交仓库 `oj/io/SubmissionStore.java`(对象流)

```java
package oj.io;
import oj.core.Submission;
import java.io.*;
import java.util.*;
public class SubmissionStore {
    private static final String DIR = "submissions";

    public static void save(Submission s) throws IOException {
        new File(DIR).mkdirs();
        try (ObjectOutputStream oos = new ObjectOutputStream(
                 new FileOutputStream(new File(DIR, s.getId() + ".ser")))) {
            oos.writeObject(s);                               // 序列化写出
        }
    }

    public static List<Submission> loadAll() throws IOException, ClassNotFoundException {
        List<Submission> list = new ArrayList<>();
        File[] files = new File(DIR).listFiles((d, name) -> name.endsWith(".ser")); // Lambda 过滤器
        if (files == null) return list;
        for (File f : files) {
            try (ObjectInputStream ois = new ObjectInputStream(new FileInputStream(f))) {
                list.add((Submission) ois.readObject());      // 反序列化读回
            }
        }
        return list;
    }
}
```

## 4. 串起来(Main 演示)

```java
Problem p = ProblemLoader.load(1);                  // 从文件读题
Submission sub = new Submission(p.getId(), "alice");
sub.setResult(new StandardJudge().judge(p, new AplusB()));
SubmissionStore.save(sub);                          // 落盘
System.out.println("历史提交数: " + SubmissionStore.loadAll().size());  // 重启后仍在
```

## 验收标准

- [ ] 在 `problems/` 新建一个题目目录就能被 `ProblemLoader.load` 读到,**不改代码**。
- [ ] 提交后 `submissions/` 出现 `<id>.ser`;重启程序 `loadAll()` 能读回历史。
- [ ] 用到了:`File` 遍历、`BufferedReader` 读文件、`ObjectOutputStream/ObjectInputStream` 序列化。
- [ ] 能解释为什么 `Solution` 不该被序列化(只存数据,不存逻辑)。
