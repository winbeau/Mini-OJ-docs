## 1. 浮点判题器 `SpecialJudge`(继承 AbstractJudge + 正则 + Math)

`oj/judge/SpecialJudge.java`
```java
package oj.judge;
public class SpecialJudge extends AbstractJudge {
    private final double eps;
    public SpecialJudge()           { this(1e-6); }     // 构造重载 + this()
    public SpecialJudge(double eps) { this.eps = eps; }

    @Override
    protected boolean compare(String expected, String actual) {
        if (actual == null) return false;
        String[] e = expected.trim().split("\\s+");      // 正则空白分隔
        String[] a = actual.trim().split("\\s+");
        if (e.length != a.length) return false;
        for (int i = 0; i < e.length; i++) {
            double d1 = Double.parseDouble(e[i]);        // String→基本类型
            double d2 = Double.parseDouble(a[i]);
            if (Math.abs(d1 - d2) > eps) return false;   // Math.abs 容差
        }
        return true;
    }
}
```

## 2. 格式校验 → PE(Pattern/Matcher)

判题前先用正则校验输出格式,不合法记 `PE`。可做成一个小工具:
```java
package oj.judge;
import java.util.regex.Pattern;
public class FormatChecker {
    // 例:要求输出是"若干个用单空格分隔的整数"
    private static final Pattern INTS = Pattern.compile("^-?\\d+( -?\\d+)*$");
    public static boolean isIntLine(String s) {
        return INTS.matcher(s.trim()).matches();         // Matcher.matches
    }
}
```
> 用法:`AbstractJudge.compare` 的子类里,先 `if(!FormatChecker.isIntLine(actual)) return false;`,并在 detail 标注 PE。PE 属增强,**可选**。

Solution bigAdd = input -> {

## 5. 反射:判题器工厂(本章亮点)

`oj/judge/JudgeFactory.java`——按**类名字符串**造判题器,判法变成配置:
```java
package oj.judge;
public class JudgeFactory {
    public static Judge create(String className) {
        try {
            Class<?> clazz = Class.forName(className);                 // 反射加载类
            return (Judge) clazz.getDeclaredConstructor().newInstance(); // 反射造对象
        } catch (Exception e) {
            throw new IllegalArgumentException("无法创建判题器: " + className, e);
        }
    }
}
```
用法:
```java
Judge j = JudgeFactory.create("oj.judge.SpecialJudge");   // 改字符串就能换判法
```
> 题目可带一个 `judgeClass` 字段(String),不同题用不同判题器——为 M5b 入库 / M6 网络传判法名做铺垫。

## 6. 判题报告(StringBuilder)

```java
public static String report(Problem p, JudgeResult r) {
    StringBuilder sb = new StringBuilder();
    sb.append("题目: ").append(p.getTitle()).append('\n')
      .append("结果: ").append(r).append('\n');            // 复用 toString
    if (!r.isAccepted()) sb.append("详情: ").append(r.getDetail()).append('\n');
    return sb.toString();
}
```
